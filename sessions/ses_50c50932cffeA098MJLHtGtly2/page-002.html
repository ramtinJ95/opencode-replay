## Options to Fix

### Option 1: Update package.json and leave the release as-is (simplest)
1. Update package.json to 1.2.0
2. Commit and push
3. The release tag stays pointing to the old commit, but future users pulling from main will have the correct version

Downside: The tagged release commit has mismatched version

### Option 2: Delete the release, update, then re-release (cleanest)
1. Delete the v1.2.0 release on GitHub (Releases → v1.2.0 → Delete)
2. Delete the tag: git push origin --delete v1.2.0
3. Update package.json to 1.2.0
4. Commit and push
5. Re-create the release via GitHub UI

Downside: More steps, but results in a clean release

### Option 3: Move the tag after updating (advanced)
1. Update package.json to 1.2.0
2. Commit and push
3. Delete old tag and create new one pointing to latest commit
4. GitHub release automatically updates to new tag location
